<h3>6-2. 문제를 해결했던 부분</h3>

🌩문제 상황🌩<br>
다른 리스트들과 다르게 관리자 메인 페이지에서는 리스트들을 5개씩 뿌려야 하기에 5개 이상씩 출력되는 부분이 발생하였습니다. <br>

타임리프 문법으로 모델 객체의 담은 것들을 foreach문으로 출력하고자 하니 break를 걸 수 없어서 데이터들이 전부 출력되었습니다. <br>
🚀해결 방법🚀<br>
타임리프 문법이 아닌 JS로 각각의 모델에 담긴 데이터들을 forEach문으로 돌린 후 count라는 변수를 선언하여 한번 출력이 될 때마다 count를 1씩 증가시켜 count의 변수가 5가 되었을시 
forEach문을 탈출 하도록 설계하였습니다.

<h3>6-3. 총평</h3>
<h4>🌟 기획은 개발의 시작점이자 대들보이다.  </h4>
개발에는 끝이 없다는 사실을 알게 되었습니다. 처음 기획 시 세부 적인 사항까지 고려하여 DB와 기능 설계를 하였지만 프로젝트가 진행되는 동안 테이블과 기획이 조금씩 수정되는 모습을 보며
많이 부족함을 느꼈습니다. 개발 단게 중 가장 많은 시간과 노력을 투자해야하는 부분이 기획이라는 것을 느꼈고 완벽한 기획일 수록 개발이 쉽게 진행되며 프로젝트의 완성도가 올라 간다는 것을
많이 느끼고 배웠습니다. 앞으로는 팀원들 과의 소통을 통해 보다 세부적이고 명확한 기획을 통해 더욱 발전된 프로젝트를 완성시킬 것입니다.


<h4>🌟🌟 소통은 해도해도 모자르다 </h4>
팀원들과의 소통에는 자신있었습니다. 그러나 사이의 원만함과 빈도수와는 별개로 프로젝트 진행 시 나의 작업 상황 진행과 오류 발생 부분, 백업을 어디까지 했는지 등 상세하고 자세하게 공유해야 한다는 것을 알게 되었습니다. 예를 들어 프로젝트 작업 중 깃허브에서 충돌이 발생하여 브랜치 생성 시 이 내용까지 공유하지 않으면, 다른 팀원들의 코드에도 영향을 미칠 수 있으며, 이 사실을 모르는 팀원이 직접 해결하기까지는 적지 않은 시간이 소요된다는 것을 알게 되었습니다. 따라서 프로젝트 종료 후 팀원들과 프로젝트 리뷰 시간을 가지고 다음 프로젝트 때 부족했던 점과 보완할 점을 의견을 나누며 다음 프로젝트에서 건설적인 방향으로 나아갈 수 있도록 하였습니다. 


<h4>🌟🌟🌟 한 발짝 더 나아가는 개발자! </h4>
이번 프로젝트에서는 관리자 페이지를 맡아서 많은 기능들을 구현 시켜야 함에 걱정이 앞섰습니다. 하지만 프로젝트 진행하면서 막히는 부분이 있으면 적극적으로 팀원들과 의견을 나누고 문제를 해결하고자 노력하여 나의 담당 백엔드를 완성 시킬 수 있었습니다. 앞으로는 경험했던 기능들은 까먹지 않게 복습하며 공부하고 담당이 아니여서 써보지 못했던 소셜 로그인 API 등을 사용해 더 많은 경험을 쌓아 지금보다 더 성장하며 지속적인 학습과 발전을 추구하며 개발자로서의 능력을 더욱 향상시키고 싶습니다. 도전하는 것에 두려움을 느끼지 않고 당당히 도전을 향해 뛰어가는 개발자가 될 것입다.
